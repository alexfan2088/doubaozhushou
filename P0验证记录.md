# 豆包助手 V2 P0 验证记录

## 当前状态

- 状态：官方 SDK 云端链路和目标手机真机闭环已完成，进入功能完善与稳定性验证。
- V1 影响：V1 兼容功能继续保留；V2 使用独立 Activity、会话客户端和前台服务。
- 云端连接：已完成，支持连续对话、用户打断、暂停恢复、设备变化和电话后重连。

## P0 目标

1. 不依赖豆包 App 建立端到端实时语音会话。
2. 通过 Newline NP-S2201 完成至少 10 轮连续对话。
3. 通过已选蓝牙通话设备完成至少 10 轮连续对话。
4. 验证模型回答期间的用户打断。
5. 测量建连、首包、打断和重连时间。
6. 确认单连接最长时长、静音计费、并发和错误码。
7. 决定正式采用官方 Android SDK还是 WebSocket。

## 云端联调所需资料

以下参数应来自“豆包端到端实时语音大模型”测试应用，而不是主账号永久 AK/SK：

- AppID。
- AppKey（旧版控制台显示为 Secret Key）。
- 测试 Access Token 或官方 Android SDK要求的授权材料。
- Resource ID：`volc.speech.dialog`，无需从控制台另行查找。
- 模型版本当前固定为 `1.2.1.1`，P0 后续可切换 `2.2.0.0` 比较。
- Voice ID 可选；P0 先使用模型公共默认音色，暂不使用未训练的复刻音色。
- 账号已开通的并发数和试用/付费额度。

凭证要求：

- 使用低额度、可撤销的测试凭证。
- 不通过聊天、Git、截图或日志提交永久 AK/SK。
- 本地配置写入已被 `.gitignore` 排除的 `local.properties`。

## SDK 文档结论与待真机验证

- 允许外部 PCM 输入：16 kHz、单声道、16-bit PCM，SDK 可重采样。
- 允许播放器音频和解码后原始音频回调。
- 支持客户端打断、暂停/恢复录音及暂停/恢复播放。
- 使用 SDK 内置录音和播放时必须开启 AEC。
- SDK 内部采播是否遵循 Android `setCommunicationDevice`。
- 服务端 VAD 和判停参数。
- 用量回调和客户端准确计费统计方式。
- 单连接最长时间及推荐重连方式。
