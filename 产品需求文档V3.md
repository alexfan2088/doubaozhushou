# 豆包助手 V3 本地 DeepSeek 语音版需求文档

## 1. 目标

V3 在 Android 手机内完成语音识别、DeepSeek 推理和语音合成。模型下载完成后不依赖
云端服务，不产生云端 Token 费用。V3 是串联式本地语音方案，不宣称为端到端语音模型。

V1、V2 的会话、配置、统计和音频服务保持不变。应用冷启动及手机重启后仍默认进入 V2。

## 2. 用户体验

- V1、V2、V3 页面均可切换到另外两个版本。
- V3 首次进入时检测模型；未安装时引导用户到设置页下载。
- 用户可选择 DeepSeek 1.5B 或 7B。程序依据内存和存储给出推荐，但最终由用户决定。
- 两种模型可同时保留，同一时间只加载一个，可分别下载和删除。
- 下载完成后，连接 Type-C、蓝牙 HFP 或使用手机麦克风即可持续对话。
- “允许播放时语音打断”默认开启，也可关闭为半双工。
- V3 显示模型、实际音频路由及聆听、识别、思考、播放、暂停和错误状态。

## 3. 模型与资源

| 模型 | 用途 | 下载大小 | 建议 |
|---|---|---:|---|
| DeepSeek-R1-Distill-Qwen-1.5B Q4_K_M | 本地文本推理 | 约 1.12GB | 速度优先 |
| DeepSeek-R1-Distill-Qwen-7B Q4_K_M | 本地文本推理 | 约 4.7GB | 质量优先 |
| Sherpa-ONNX Streaming Paraformer INT8 | 中英文流式 ASR | 约 237MB | 两种 LLM 共用 |
| 内置 VITS 中文模型 | 本地 TTS | 随 APK 安装 | 五种音色 |

推荐 7B 必须同时满足：总内存不少于 12GB、加载前可用内存不少于 6GB、可用存储
不少于 10GB。否则推荐 1.5B。存储不足时禁止下载；性能风险只提示，不替用户改变选择。

## 4. 对话行为

- 静音 700ms 结束一句，单句最长 15 秒。
- 默认回答不超过两句话，最多生成 128 Token。
- `<think>` 推理内容不显示、不朗读，只展示并播放最终回答。
- 默认保留最近四轮上下文，达到设置上限后重建本地上下文。
- 打断开启时，播放期间继续本地识别；确认用户说话后停止 TTS 和未完成生成，并以新语音
  开始下一轮。
- 打断关闭时，播放期间暂停识别，播放完成后恢复。
- 电话或暂停时停止录音和播放但保留已加载模型，以便快速恢复；版本切换时释放 V3
  录音、播放、模型、Audio Focus 和蓝牙通信路由。

## 5. 验收标准

- 默认入口始终为 V2，V1/V2 回归测试通过。
- 下载支持断点续传、文件大小及 SHA-256 校验，失败文件不能标记为已安装。
- 飞行模式下完成至少五轮 V3 中文语音对话。
- 手机麦克风、蓝牙 HFP、Type-C 拾音和手机麦克风回退均能正确显示。
- 默认打断可停止播放并进入新一轮；关闭打断后不得在播放期间识别。
- 连续运行 30 分钟无崩溃、并行重复推理、串音或版本切换后的模型残留。
