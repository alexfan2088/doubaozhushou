# 豆包助手 V3 本地 DeepSeek 语音版技术方案

## 1. 隔离边界

V3 代码位于独立 `v3` 包，使用 `v3_settings`、`v3-models`、V3 前台服务和独立蓝牙
选择文件。V1/V2 只增加导航按钮。V3 不调用火山客户端、不写 V2 用量统计，也不启动
V1 无障碍服务。

## 2. 数据链路

```text
AudioRecord 16k PCM
  -> AcousticEchoCanceler / NoiseSuppressor
  -> Sherpa-ONNX Streaming Paraformer INT8
  -> llama.cpp Android / DeepSeek GGUF
  -> 过滤 <think>
  -> 分句
  -> 内置 VITS TTS
  -> 当前通信音频路由
```

ASR 使用 `VOICE_COMMUNICATION` 音源、100ms 音频帧和 Sherpa endpoint。DeepSeek 使用
llama.cpp 官方 Android 绑定的 ARM64 CPU 后端。1.5B 使用 4 线程，7B 使用 6 线程；
上下文 2048、最大输出 128 Token。CPU 后端按设备指令集动态加载，因此 APK 使用
解压式 JNI 打包。

## 3. 模型管理

- LLM 从 ModelScope 下载固定的 QuantFactory Q4_K_M 文件。
- ASR 从 Sherpa 官方 Hugging Face 仓库下载，优先 `hf-mirror.com`，官方地址回退。
- 下载写入 `.part`，通过 HTTP Range 续传；完成后校验固定文件大小和 SHA-256，再原子改名。
- 1.5B 和 7B 使用独立文件，公共 ASR 只保存一份。
- 资源推荐读取 `ActivityManager.MemoryInfo` 和 `StatFs`，只影响提示，不覆盖用户选择。

## 4. 状态机与打断

主要状态为：未安装、下载、校验、模型加载、聆听、识别、思考、播放、暂停、错误。
同一时刻只允许一个 DeepSeek 生成任务。

打断开启时 ASR 在播放期间继续运行。系统 AEC/NS 优先降低回采；ASR 结果与当前播放文本
高度相似时视为回声。确认新语音后同时取消 llama token 流和 TTS 队列，保留已完成历史，
进入新一轮。打断关闭时直接暂停 ASR 数据处理。

## 5. 生命周期与测试

- 电话、暂停和设备切换停止录音、播放及生成任务，但保留模型以便快速恢复。
- 版本切换必须停止 V3 前台服务、AudioRecord、TTS、DeepSeek 和蓝牙 SCO。
- 单元测试覆盖默认配置和模型推荐阈值。
- 真机测试覆盖模型下载恢复、离线推理、三类音频路由、打断开关、30 分钟稳定性及 V1/V2
  回归。
